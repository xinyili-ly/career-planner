# 基于AI的大学生职业规划智能体 - 技术实施方案

## 1. 赛题深度解析

### 1.1 核心目标
构建一个智能体系统，能够连接“企业岗位需求”与“学生个人能力”，解决大学生就业中的信息不对称和认知模糊问题。

### 1.2 关键痛点与解决方案
| 痛点 | 解决方案 | 技术关键点 |
| :--- | :--- | :--- |
| **自我认知模糊** | **学生能力画像构建** | 简历解析、多维能力打分模型 |
| **信息不对称** | **岗位画像与图谱** | 岗位数据清洗、RAG检索、知识图谱构建 |
| **规划落地难** | **人岗匹配与路径规划** | 差异化分析算法、路径规划算法、动态报告生成 |

### 1.3 核心指标
*   **数据量**：处理 10,000 条岗位数据。
*   **图谱要求**：构建垂直晋升与水平换岗路径（至少 5 个岗位，每岗 2 条路径）。
*   **准确率**：画像关键信息准确率 > 90%，人岗匹配准确率 > 80%。

---

## 2. 系统总体架构

采用 **前后端分离** 架构，后端以 Python 为核心进行 AI 逻辑处理。

*   **前端层 (User Interface)**: React / Vue 或 Streamlit (快速原型)。负责简历上传、交互式问答、报告展示。
*   **应用层 (Backend API)**: FastAPI。负责业务逻辑调度、Prompt 管理、Session 管理。
*   **智能层 (AI Agent Core)**:
    *   **LLM**: DeepSeek-V3 / GPT-4 (语义理解、推理、生成)。
    *   **RAG Engine**: LangChain / LlamaIndex (文档检索)。
    *   **Matching Engine**: 自研多维匹配算法。
*   **数据层 (Data Infrastructure)**:
    *   **Vector DB**: Chroma / Milvus (存储岗位/简历向量，用于检索)。
    *   **Graph/Relational DB**: Neo4j 或 PostgreSQL (存储岗位关系图谱、用户数据)。
    *   **Object Storage**: 本地文件系统或 OSS (存储原始简历、生成的 PDF 报告)。

## 3. 分步实施方案与技术细节

### 第一阶段：数据基础设施建设 (岗位画像与图谱)

**目标**：处理 10,000 条岗位数据，构建“岗位知识库”和“职业路径图谱”。

*   **任务 1.1：数据清洗与结构化**
    *   **做什么**：将原始非结构化 JD (职位描述) 转化为结构化 JSON (技能、学历、经验、软素质)。
    *   **方法**：使用 LLM 编写 `Extraction Prompt`，批量处理文本。
    *   **技术**：Python (Pandas), LangChain, LLM (DeepSeek)。

*   **任务 1.2：构建岗位画像向量库**
    *   **做什么**：将结构化后的岗位需求向量化，以便后续进行语义搜索。
    *   **方法**：使用 Embedding 模型 (如 bge-m3 或 text-embedding-3) 将岗位描述转为向量存入数据库。
    *   **技术**：ChromaDB / Faiss。

*   **任务 1.3：构建岗位关联图谱 (难点)**
    *   **做什么**：建立岗位的“垂直晋升”和“横向转岗”关系。
    *   **方法**：
        1.  利用 LLM 分析岗位 Title 和职责，推断层级关系 (如：助理 -> 专员 -> 经理)。
        2.  计算岗位间的技能重合度，重合度高但领域不同的定义为“换岗路径”。
        3.  使用图数据库或简单的 Adjacency List 存储关系。
    *   **技术**：Neo4j (推荐) 或 NetworkX (轻量级), Cypher 查询语言。

### 第二阶段：学生能力画像构建 (输入端)

**目标**：精准解析学生简历，生成能力雷达图。

*   **任务 2.1：简历解析与多模态输入**
    *   **做什么**：支持 PDF/Word 上传，提取文本。
    *   **方法**：使用 OCR 或文档解析库提取纯文本。
    *   **技术**：PyPDF2, python-docx, PaddleOCR (针对图片简历)。

*   **任务 2.2：能力提取与评分**
    *   **做什么**：从简历中提取 7 大维度 (专业技能、证书、创新、学习、抗压、沟通、实习)。
    *   **方法**：设计 `Student Profiling Prompt`，让 LLM 输出 JSON 格式，并根据关键词密度和描述质量进行初步评分 (1-100)。
    *   **技术**：Pydantic (数据验证), LLM。

### 第三阶段：核心匹配引擎与路径规划 (智能层)

**目标**：实现人岗精准匹配，输出差距分析。

*   **任务 3.1：多维人岗匹配算法**
    *   **做什么**：计算学生与目标岗位的匹配度。
    *   **方法**：
        1.  **硬性门槛过滤**：学历、专业 (Rule-based)。
        2.  **语义匹配**：计算简历向量与岗位向量的 Cosine Similarity。
        3.  **加权打分**：按赛题要求 (基础要求 20%, 职业技能 40%, 职业素养 20%, 发展潜力 20%) 进行加权计算。
    *   **技术**：Scikit-learn (计算相似度), NumPy。

*   **任务 3.2：差距分析 (Gap Analysis)**
    *   **做什么**：明确指出“缺什么证书”、“缺什么技能”。
    *   **方法**：将学生画像集合与岗位画像集合取差集 (Set Difference)，利用 LLM 生成自然语言解释。

### 第四阶段：职业生涯发展报告生成 (输出端)

**目标**：生成可执行、可编辑的报告。

*   **任务 4.1：报告生成 Agent**
    *   **做什么**：综合上述信息，撰写短期/中期/长期计划。
    *   **方法**：使用 Chain-of-Thought (CoT) Prompting，让 LLM 扮演职业规划师，分章节生成内容。
    *   **技术**：Jinja2 (模板引擎), LLM。

*   **任务 4.2：文档导出**
    *   **做什么**：将 Markdown 或 HTML 报告转为 PDF/Word。
    *   **技术**：ReportLab, python-docx, wkhtmltopdf。

### 第五阶段：交互界面与系统集成

**目标**：提供友好的用户界面。

*   **任务 5.1：开发 Web 界面**
    *   **做什么**：简历上传入口、聊天对话框、图谱可视化展示、报告在线编辑。
    *   **技术**：FastAPI (后端), React/Streamlit (前端), ECharts/D3.js (图谱可视化)。

## 5. 赛题加分项策略

1.  **图谱可视化**：不要只做文字描述，前端必须用动态图表展示“垂直”和“横向”路径。
2.  **真实数据验证**：在开发过程中，找几份真实的计算机专业学生简历进行测试，微调 Prompt，确保“建议”不是泛泛而谈，而是具体的（例如：“建议考取 AWS 助理架构师证书” 而不是 “建议考取相关证书”）。
3.  **动态反馈**：允许用户在对话中修正自己的画像（例如：“我其实还有个大厂实习没写在简历里”），Agent 应能即时更新画像并重新匹配。

## 6. 技术概念详解：向量 (Vector) 与 Embedding

### 6.1 什么是“向量”？
在 AI 和自然语言处理 (NLP) 领域，**向量 (Vector)** 指的是将一段文本（如“Java开发工程师”或“熟悉Python编程”）转换成的一串**数字列表**（例如 `[0.12, -0.98, 0.55, ...]`）。

*   **传统匹配**：基于关键词。如果岗位要求“软件工程师”，而简历写的是“程序员”，计算机可能认为不匹配。
*   **向量匹配**：基于**语义**。计算机通过算法知道“软件工程师”和“程序员”在语义空间中距离很近，因此认为它们是高度匹配的。

这个将文本转化为向量的过程称为 **Embedding (嵌入)**。

### 6.2 为什么要计算“简历向量”与“岗位向量”？
通过计算两个向量之间的距离（通常使用余弦相似度 Cosine Similarity），我们可以量化地判断一份简历与一个岗位的匹配程度，而不仅仅是看关键词是否重合。

### 6.3 如何生成向量？
生成向量需要使用专门的 **Embedding 模型**。

**具体步骤：**

1.  **选择模型**：
    *   **商业模型**：OpenAI `text-embedding-3-small` / `text-embedding-3-large`。
    *   **开源模型**（推荐）：`BAAI/bge-m3` (智源 BGE 模型，对中文支持极佳) 或 `m3e-base`。

2.  **调用接口/代码生成**：
    将文本传入模型，模型会返回一个固定长度的数组（例如 1024 维或 1536 维）。

**代码示例 (Python + LangChain/Sentence-Transformers):**

```python
from sentence_transformers import SentenceTransformer

# 1. 加载模型 (自动下载)
model = SentenceTransformer('BAAI/bge-m3')

# 2. 准备文本
resume_text = "熟悉 Python，有 3 年后端开发经验，精通 FastAPI。"
job_text = "招聘 Python 后端工程师，要求熟悉 Web 框架。"

# 3. 生成向量
resume_vector = model.encode(resume_text)
job_vector = model.encode(job_text)

# 4. 向量长什么样？
# 它是一个包含很多浮点数的数组，例如：[-0.012, 0.543, -0.231, ...]
print(f"向量维度: {len(resume_vector)}") 
```

3.  **存储与使用**：
    *   生成的向量通常存储在 **向量数据库** (Vector DB) 如 ChromaDB 或 Milvus 中。
    *   当需要匹配时，直接在数据库中搜索“与该简历向量距离最近的岗位向量”。
